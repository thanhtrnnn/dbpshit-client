Table: Customer (already exists)

Assume there is an existing table Customer with:

- CustID as the primary key,

- basic customer information such as name, email, city, etc.


You must create the SaleOrder table so that it references this existing Customer table.


Table to create: SaleOrder

+--------------+--------------------------------+
| Column Name  | Type                           |
+--------------+--------------------------------+
| OrderID      | int                            |
| CustID       | int                            |
| OrderDate    | date                           |
| Status       | enum('NEW','PAID','CANCELLED') |
| TotalAmount  | decimal(10,2)                  |
+--------------+--------------------------------+

OrderID is the unique ID of each order and will be the primary key.  

CustID is the ID of the customer who placed the order; it must reference CustID in the existing Customer table.  

OrderDate is the date when the order was created.  

Status indicates the current order status and uses an ENUM with values 'NEW', 'PAID', or 'CANCELLED'.  

TotalAmount is the total value of the order.  

Each row in this table represents a single order placed by a customer.


Task:  

Based on the existing Customer table, write an SQL statement to create only the SaleOrder table with:

- OrderID as the primary key,

- a foreign key from SaleOrder.CustID to Customer.CustID,

- Status defined as ENUM('NEW','PAID','CANCELLED'),

- appropriate NOT NULL constraints as described.
